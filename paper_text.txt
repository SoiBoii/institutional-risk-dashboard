Abstract. Increased reliance on computer technology led to the emer-
gence of secure password storage as an important issue, with the diffi-
culty of finding a balance between computing efficiency and the ability
to resist modern-day attacks. The task remains hard because previously
known methods were unable to provide both. Several algorithms exist to-
day including SHA-256, bcrypt, PBKDF2, and Argon2, yet none of them
combines all desired properties. SHA-256 is one of the most computation-
ally efficient methods yet lacks any attack resistant property. The next
two methods increase attack resistance but at the expense of adding ad-
ditional computational effort. Furthermore, neither algorithm possesses
memory hardness, which is essential in order to withstand hardware at-
tacks. All that makes it necessary to develop a comprehensive index of
evaluation of different password hashing algorithms. Currently, methods
for analysis of various passwords are limited to specific factors, such as
speed or cost. As opposed to that, the proposed Security Efficiency In-
dex (SEI) integrates all aspects, including resistance to dictionary, brute
force attacks, memory hardness, and computational delay. Experiments
show that even though SHA-256 is highly efficient yet highly vulnerable
to brute force attacks, Argon2 outperforms other algorithms when tested
according to various SEI factors.

Introduction
Password authentication is the most commonly employed mode of verifying users
on the Internet, within various software applications, within organizational in-
tranets, within banking applications, and within cloud computing. EPassword
authentication is the most commonly employed mode of verifying users on the

within banking applications, and within cloud computing. Even with improve-
ments like multi-factor authentication and security tokens, the usage of pass-
words has not decreased and passwords continue to be the main and sometimes
the only means of authentication [7]. In recent times, massive credential breaches
in the last decade have consistently demonstrated the devastating impact of in-
adequate password security measures. Breaches involving hundreds of millions
of unsalted or weakly hashed credentials from prominent organizations have
demonstrated the gap between password security theory and practice [5]. In the
case of exfiltrated password databases, the security of the credentials depends
on the computational and memory costs imposed by the hashing algorithm.
Modern attackers exploit GPU acceleration and parallel architectures, making
memory-hard hashing essential [4]. GPU-based optimizations significantly accel-
erate password cracking processes [11]. Advanced parallel cracking techniques
further amplify attack efficiency [12]. If algorithms are not capable of imposing
a high enough adversary cost, it can result in rapid cracking even after the detec-
tion and remediation of the initial breach. Recent large-scale credential breaches
have demonstrated weaknesses in password security practices [7]. Probabilistic
password cracking models further expose vulnerabilities in real-world password
datasets [13]. Deep learning-based password guessing techniques further enhance
adversarial capabilities in practical attack scenarios [16].
1.1
Research Gap
Nevertheless, in the presence of a number of adaptive KDFs, there exist a num-
ber of gaps in the password hashing literature. In most of the comparative pass-
word hashing literature, benchmarking has been performed on a single parame-
ter, i.e., execution time/throughput, without considering memory consumption,
brute force scalability, and use of system resources in an integrated manner. This
is because most of the password hashing literature has followed a reductionist
approach, which is not enough in understanding the multidisciplinary security
of password hashing algorithms [1]. In addition, multidisciplinary parametric
sensitivity analysis of password hashing algorithms, in which memory cost, iter-
ation count, and parallelism are varied in an integrated manner, has not been
addressed properly in the password hashing literature. In most guidelines, only
minimum values of parameters are provided. However, there exists a lack of
quantitative characterization of parameter space. Recent studies highlight the
absence of unified frameworks integrating both performance and security met-
rics [3]. Security-performance trade-offs in cryptographic systems require more
comprehensive evaluation models [10].

Proposed Evaluation Framework
A systematic analysis of password hashing functions needs an evaluation scheme
that considers security and efficiency aspects together. Existing methods based
on individual parameters like computation time or bandwidth fall short when it
comes to dealing with contemporary attack strategies. Offline attacks on compro-
mised credentials are enabled by parallel processing and memory architectures,
making it clear that a new approach is needed. In order to assess the efficiency of
a password hash function, a framework for evaluation must be devised to measure
both security and efficiency aspects of the algorithms. Traditional metrics such
as runtime and throughput fail to serve this purpose, as contemporary adver-
sary models utilize modern capabilities of parallelization and memory to launch
attacks against passwords. Such approaches fail to accurately reflect adversarial
capabilities in modern environments [15] Such approaches fail to accurately re-
flect adversarial capabilities in modern environments [2]. In order to meet this
challenge, a framework is developed to evaluate algorithms based not only on
their empirical runtime but also on a simulated adversary cost model. The pro-
posed framework evaluates algorithms under various settings, such as different
numbers of iterations, memory size, and level of parallelization. In particular,
the measurement of runtime characteristics is performed through analysis of an
algorithm’s latency, memory usage, and scalability.

Security Efficiency Index (SEI)
The Security Efficiency Index (SEI) forms the heart of the above framework and
is essentially an index that encapsulates the relationship between security and
efficiency. SEI takes into account three important parameters, namely:

This equation embodies the idea of increasing the cost of an attack while
retaining functionality. Multiplication of B and M accounts for the overall cost
to an attacker in terms of computing and memory, and division by T ensures
that highly inefficient setups are not rewarded. This formulation aligns with
modern cryptographic practices that combine computational and memory costs
for secure password hashing [4]. This formulation is also supported by adversarial
cost modeling approaches [2].

metrics are captured for execution time, memory usage, and CPU consumption.

conditions. The benchmarking framework was written in Python, using the
hashlib library for SHA-256, the bcrypt library for bcrypt, the argon2-cffi
library for Argon2, and the passlib library for PBKDF2. The system resource
utilisation of the password hashing algorithms’ memory consumption was mea-
sured by the RSS. The CPU utilisation of the algorithms was measured imme-
diately after each run.

Comparative Multi-Metric Analysis
To measure the system’s performance in various criteria, radar charts compar-
isons were carried out among the best-performing systems based on SEI. Metrics
considered include execution time, memory usage, throughput, and SEI.
All metrics were scaled using min-max normalization, which enabled com-
parison across different units. The obtained results reveal that Argon2 configu-
rations requiring high memory usage, such as Argon2 (m = 65536, t = 1, p = 1)
and (m = 32768, t = 1, p = 1), perform better across all criteria.
The reason is that they make full use of memory usage and produce the high-
est SEIs while at the same time taking less execution time. While the throughput
might not be as good as fast hashing algorithms such as SHA-256, it is satisfac-
tory enough for authentication purposes.

Conclusion
Estimation of password hashing algorithm strength implies the existence of an
all-encompassing characteristic that would reflect both the performance charac-
teristics and the security provided by the algorithm. One such multi-parameter
indicator is the concept of Security Efficiency Index (SEI) that combines the
resistance against brute force attack, memory hardness, and latency in order
to create one metric for evaluation. Benchmarking and statistical analysis have
allowed to draw clear lines between different algorithms. Argon2 was found to
provide the maximum SEI since it could make use of memory alongside being
flexible in the amount of computational power, thus greatly increasing the dif-
ficulty for potential attackers. On the contrary, SHA-256 showed itself to be
computationally efficient but lacking necessary resilience to brute force attacks;
thus, it cannot be recommended for use in storing password hashes safely. While
bcrypt and PBKDF2 increase security thanks to increased computational work
factor, their limited memory hardness makes them less efficient than other algo-
rithms in terms of SEI. Thus, memory utilization proved to be an even stronger
contributor to security enhancement than increasing the iteration count. Argon2
can be seen as a solution that manages to balance the strength and performance
aspects of password hashing successfully [4]. These findings are consistent with
broader analyses of security-performance trade-offs in password hashing sys-
tems [10]. Recent security guidelines also reinforce the importance of robust
password hashing practices [7].
